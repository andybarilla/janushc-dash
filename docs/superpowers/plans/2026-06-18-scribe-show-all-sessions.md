# Scribe Show All Sessions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show and count every scribe session awaiting review by removing the fixed backend list cap and simplifying the patient column copy.

**Architecture:** Keep the existing scribe session list endpoint and frontend query flow. Change the SQL source, regenerate sqlc output, and update the existing inbox table presentation without adding pagination, filters, schema changes, or summary endpoints.

**Tech Stack:** Go 1.25+, chi, sqlc, SQLite, React 19, Vite, TypeScript, Tailwind CSS.

## Global Constraints

- Backend `ListScribeSessions` must return all matching sessions for the tenant, ordered newest first.
- Regenerate `internal/database/scribe.sql.go` with `make sqlc`; do not hand-edit generated sqlc output.
- Frontend table header must read `Patient`.
- Frontend patient column must render `entry.session.patient_id` only.
- Out of scope: pagination, filtering changes, schema changes, and new summary endpoints.
- Verification commands: `go test ./...` and `cd frontend && npm run build`.
