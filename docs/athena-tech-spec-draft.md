## Workflow Description (User Workflow)

**User:** physician (provider) at the practice.

**Ambient Scribe → Encounter Note workflow**
1. Provider logs into the Janus dashboard (JWT auth).
2. The app identifies the visit to document — the provider's encounter for a given patient — using athenahealth appointment/encounter data.
3. During/after the visit, audio is transcribed and summarized by AI into structured note sections (HPI, Assessment & Plan, Physical Exam).
4. The provider reviews and edits the generated note in the Janus UI.
5. On finalize, the app writes the approved note sections back to the corresponding athenahealth encounter.
6. The provider completes the encounter in athenaOne as usual. Workflow ends when the note is written.

No SSO into athenaNet. No legacy-system data import. No bulk/large data pulls (see cadences below).

## Use Cases

> Read paths verified against the sandbox in `docs/athena-api-audit.md`. The note-write endpoints (UC3) are NOT yet verified — confirming them is the critical path for this integration.

### Use Case #1 — Identify the provider's encounters to document
| Action | Cadence | API Endpoint |
|---|---|---|
| Get booked appointments for a department (narrow window, ≤ today/next day) | On dashboard load, per provider session | `GET /v1/{practiceid}/appointments/booked?departmentid={id}&startdate={d}&enddate={d}` |
| List encounters for the patient to resolve the encounter to write to | On demand per patient being documented | `GET /v1/{practiceid}/chart/{patientid}/encounters?departmentid={id}` |
| Resolve patient name/details | Per patient being documented | `GET /v1/{practiceid}/patients/{patientid}` |

### Use Case #2 — Reference data (setup / low frequency)
| Action | Cadence | API Endpoint |
|---|---|---|
| List practice departments | On setup + as needed | `GET /v1/{practiceid}/departments` |
| List providers | On setup + as needed | `GET /v1/{practiceid}/providers` |

### Use Case #3 — Write ambient-scribe encounter documentation
> VERIFIED against docs.athenahealth.com/api/workflows/adding-notes-to-an-encounter. Prerequisite: the encounter must be OPEN (patient checked in) to have a writable `encounterid`. For HPI/PE/ROS, GET the section first and echo the full returned template/array back in the PUT — omitting it deletes existing provider documentation.

| Action | Cadence | API Endpoint |
|---|---|---|
| Write Assessment & Plan (simplest — PUT only) | On provider finalize | `PUT /v1/{practiceid}/chart/encounter/{encounterid}/assessment` — body `assessmenttext` + replace flag |
| Read HPI section (preserve existing) | Immediately before each HPI write | `GET /v1/{practiceid}/chart/encounter/{encounterid}/hpi` |
| Write History of Present Illness | On provider finalize | `PUT /v1/{practiceid}/chart/encounter/{encounterid}/hpi` — body `sectionnote` + full `hpi` array from the GET |
| Read Physical Exam section (preserve existing) | Immediately before each PE write | `GET /v1/{practiceid}/chart/encounter/{encounterid}/physicalexam` |
| Write Physical Exam | On provider finalize | `PUT /v1/{practiceid}/chart/encounter/{encounterid}/physicalexam` — body `sectionnote` (+ template ID from GET) |
